# PRD: workload_logs

워크로드를 뒷받침하는 파드의 컨테이너 로그를 조회하는 도구.

## 달성 가치
- **V1: 자연어로 클러스터 운영** — 파드 이름을 모르더라도 워크로드 기준으로 로그를 조회해
  신속히 진단한다.

## 도구 개요
- 입력: `kind`·`namespace`·`name`(필수), `container`(다중 컨테이너 시 필수),
  `tail_lines`(1–5000, 기본 200), `previous`(기본 false), `timestamps`(기본 false),
  `since_seconds`(선택)
- 동작: 워크로드의 파드 셀렉터를 해석해 첫 Running 파드(없으면 매칭되는 임의 파드)의 로그 반환
- 어노테이션: `readOnlyHint=true`, `idempotentHint=true`

## Acceptance Criteria

### AC1: 워크로드 기준 로그 조회
- **설명**: 워크로드 셀렉터를 해석해 대상 파드의 컨테이너 로그를 반환한다.
- **달성 가치**: V1
- **검증 방법**: 정상 워크로드에서 최근 로그가 반환된다.

### AC2: tail 라인 제어
- **설명**: `tail_lines`로 반환 라인 수를 조절한다. 생략 시 기본 200이며, 허용 범위는 1–5000이다.
  5000을 초과하는 값은 스키마 검증으로 **거부**된다(클램프하지 않음).
- **달성 가치**: V1
- **검증 방법**: 생략 호출이 기본 200으로 동작하고, 5000 초과 요청(예: 999999)은 거부 에러를
  반환한다.

### AC3: 크래시 루프 후 직전 로그
- **설명**: `previous=true`이면 종료된 직전 컨테이너 인스턴스의 로그를 반환한다. Running 파드가
  없어도 매칭 파드를 사용하므로 크래시 루프 상황에서도 동작한다.
- **달성 가치**: V1
- **검증 방법**: 재시작된 파드에 대해 previous 로그가 재시작 이전 인스턴스의 로그를 반환한다.

### AC4: 컨테이너 선택과 필터
- **설명**: 파드에 컨테이너가 둘 이상이면 `container`가 필요하다. `timestamps`·`since_seconds`로
  출력 형식과 시간 범위를 조절할 수 있다.
- **달성 가치**: V1
- **검증 방법**: 멀티컨테이너 파드에서 container 미지정 시 거부/안내되고, timestamps·since_seconds가
  출력에 반영된다.
